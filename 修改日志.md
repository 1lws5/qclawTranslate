## 2026-07-08

### 17:05 — 技术人员 — ✅ 已实现
- **热键 Ctrl+Q/Ctrl+E 双路读选中（UIA 偏移截取优先，失败回退 Ctrl+C）**

**背景**：v1.5 切除 `_copy_selection` 后只剩纯 UIA 一条路，Edge (Chromium) 82 个 TextPattern 假阳性（全为光标幻象，lo=hi 空区间）+ Anki (Qt) 零 TextPattern，导致两个应用完全不可用。技术顾问评审指正：不应在 `_find_text_pattern_recursive` 中加 `GetText(-1)` 过滤（会误杀 Notepad），当前 `lo < hi` 判断已正确区分真假选区。

**植入**（3 个新函数 + 1 个分派器，~100 行）：
- `_read_selection(hwnd)` — 分派器：UIA 偏移截取成功→返回；失败→回退 `_copy_selection_fallback`
- `_bring_to_front_hwnd(hwnd)` — AttachThreadInput + SetForegroundWindow + BringWindowToTop（从 git `31a1ebf` 恢复）
- `_send_ctrl_c_sendinput()` — SendInput 注入 Ctrl+C（从 git `31a1ebf` 恢复）
- `_copy_selection_fallback(hwnd)` — Ctrl+C 搬运 + 剪贴板轮询（1.2s超时/30ms间隔）+ 所有分支恢复剪贴板

**调用方修改**：
- `_hotkey_translate_action`：`_read_selection_uia(hwnd)` → `_read_selection(hwnd)`
- `_hotkey_tts_action`：同上

**不动/不改**：
- `_find_text_pattern_recursive` — 保持 `sel.Length > 0` 判断，不加 GetText(-1) 过滤（技术顾问指正）
- `_read_selection_uia` — 保持 `lo < hi` 判断，逻辑正确无需改

**设计原则**（技术顾问评审意见）：
- 三种应用三条路径：Notepad → UIA 偏移截取；Edge/Anki/Qt → Ctrl+C 回退
- Ctrl+C 回退在 Edge/Anki 上可用概率高（非 UWP XAML Islands 架构，走经典 Win32 消息泵）
- 不是"临时补丁"，是"UIA 覆盖不到场景的永久 fallback"

**打包**：重新打包覆盖 exe `C:\Users\Administrator\Desktop\qclawTranslate\qclawTranslate.exe`

**待验证**：Edge / Anki 真机实测 Ctrl+C 回退链路是否可用
