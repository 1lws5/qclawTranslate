# 豆包划词与划词工具栏技术实现分析报告

**生成时间：** 2026-07-06 23:30 CST  
**目标：** 解构豆包桌面版「划词选中文本自动弹出工具栏」的实现原理，为 qclawTranslate 划词工具栏提供技术路线参考

---

## 一、用户可见行为

豆包桌面版的划词工具栏工作方式：

1. 用户在**任意应用**（浏览器、WPS、微信、VSCode、记事本、PDF阅读器等）中用鼠标选中一段文字
2. 松开鼠标后，豆包在选中文字旁**自动弹出**一个工具栏浮窗
3. 工具栏提供：搜索、翻译、解释、总结、朗读、复制、自定义技能等按钮
4. 用户可以自定义工具栏上的技能（预设 Prompt），选中文字后一键触发

关键特点：**全桌面覆盖、无需热键、鼠标划选完就自动出现**

---

## 二、核心结论

> **豆包使用 Windows 辅助功能（Accessibility）API 实现划词检测和文本读取，用一个独立渲染的弹出窗口展示工具栏。**

核心技术栈：

```
SetWinEventHook (系统级文本选择变化事件监听)
        ↓
IUIAutomation::GetFocusedElement / ElementFromPoint (获取文本容器)
        ↓
IUIAutomationTextPattern::GetSelection (读取选中文字内容)
        ↓
IUIAutomationTextRange::GetBoundingRectangles (获取选中区域屏幕坐标)
        ↓
自定义 Popup 窗口 (在坐标附近显示工具栏，无焦点抢占)
```

## 三、技术逐层拆解

### 3.1 macOS：明确使用 Accessibility API

有公开报道直接确认（来源：今日头条深度测评）：

> "在 Mac 系统中，需要开启「辅助功能权限」以实现划词时的 AI 搜索、翻译等功能。"

这表明在 macOS 上豆包使用 **AXUIElement API**（macOS Accessibility 框架），工作原理：
- `AXUIElementCopyAttributeValue(element, kAXSelectedTextAttribute, ...)` 读取选中文字
- `AXUIElementCopyAttributeValue(element, kAXSelectedTextRangeAttribute, ...)` 获取选中范围
- 通过 `kAXSelectedTextChangedNotification` 监听文本选择变化

### 3.2 Windows：系统级文本选择变化监听

Windows 的等价方案是 **SetWinEventHook**：

```c
// 注册全局文本选择变化事件钩子
HWINEVENTHOOK hook = SetWinEventHook(
    EVENT_OBJECT_TEXTSELECTIONCHANGED,  // 事件ID = 0x8014
    EVENT_OBJECT_TEXTSELECTIONCHANGED,  // 范围：仅监听这一个事件
    NULL,                                // DLL句柄（NULL = 进程外钩子）
    WinEventProc,                        // 回调函数
    0, 0,                                // 监听所有进程/线程
    WINEVENT_OUTOFCONTEXT                // 进程外回调（不需要DLL注入）
);
```

`EVENT_OBJECT_TEXTSELECTIONCHANGED` (0x8014) 是一个标准的 WinEvent，当任何支持辅助功能的窗口内文本选择发生变化时，系统会触发此事件。

**触发条件：**
- 用户在编辑框、文本框、浏览器、代码编辑器等支持 UIA TextPattern 的控件中改变选中范围
- 事件在鼠标释放、键盘方向键移动光标时都会触发
- 按 Escape 取消选中也会触发（此时所选文本为空）

### 3.3 读取选中文字：UI Automation TextPattern

事件触发后，通过 Windows UI Automation 读取选中内容：

```python
# Python 伪代码 — 对应 Windows UIA COM 调用
import comtypes.client
from comtypes.gen.UIAutomationClient import CUIAutomation8, IUIAutomationTextPattern

uia = comtypes.client.CreateObject(CUIAutomation8)

# 方法1: 从事件携带的 hwnd 获取
element = uia.ElementFromHandle(hwnd)

# 方法2: 获取当前焦点元素（备用）
element = uia.GetFocusedElement()

# 获取 TextPattern（ID=10014）
text_pattern = element.GetCurrentPattern(10014)

# 读取选中内容
selection = text_pattern.GetSelection()
for i in range(selection.Length):
    text_range = selection.GetElement(i)
    text = text_range.GetText(-1)  # -1 = 读取全部文本
    print(text)
```

**TextPattern 支持范围：**
- ✅ 原生 Win32 Edit/RichEdit 控件
- ✅ WPF / WinForms 文本框
- ✅ UWP 应用
- ✅ Chromium 内核浏览器（Chrome/Edge/豆包自身）
- ✅ VS Code (Electron)
- ✅ Notepad++、Sublime Text
- ✅ Microsoft Office (WinWord、Excel)
- ✅ Foxit Reader、Adobe Acrobat 部分版本
- ❌ 部分老旧 Win32 自绘控件（完全不暴露 UIA 接口的）
- ❌ 部分 Java Swing 应用（取决于 JRE 版本和配置）

### 3.4 获取选中文字屏幕坐标：GetBoundingRectangles

工具栏需要显示在选中文字附近，因此需要坐标：

```python
# 获取选中文字在屏幕上的矩形区域
bounding_rects = text_range.GetBoundingRectangles()
# 返回一个 IUnknown 数组，每个元素是一个 RECT (left, top, right, bottom)
# 取第一个（或计算选区中心/底部）
```

### 3.5 显示工具栏：Popup 窗口

工具栏是一个**独立的无焦点弹出窗口**：

```python
# Python/PyQt5 实现思路
class ToolBarPopup(QWidget):
    def __init__(self):
        super().__init__()
        # 关键属性：
        self.setWindowFlags(
            Qt.ToolTip |           # 无任务栏图标
            Qt.FramelessWindowHint |  # 无边框
            Qt.NoFocus             # 不抢占焦点
        )
        self.setAttribute(Qt.WA_ShowWithoutActivating)  # 显示时不激活本窗口
        # 水平布局：放置翻译、朗读、解释、总结等按钮
```

**关键点：** `WA_ShowWithoutActivating` 确保工具栏弹出时不会抢夺前台窗口的焦点，被划词的目标窗口保持激活状态。

### 3.6 工具栏自动消失

工具栏需要在一定条件下自动关闭：
- **超时**：鼠标没有移动到工具栏上，几秒后消失
- **失焦**：点击了工具栏以外的区域
- **重新划词**：用户在别处选中了新文字 → 关闭旧工具栏，打开新的
- **ESC 键**：用户按 ESC 取消选中

```python
class ToolBarPopup(QWidget):
    def __init__(self):
        # ...
        self._timer = QTimer()
        self._timer.timeout.connect(self._fade_out)

    def show_near(self, rect):
        self.move(rect.left, rect.bottom + 4)  # 选中文字下方4px
        self.show()
        self._timer.start(5000)  # 5秒后开始淡出

    def enterEvent(self, event):
        self._timer.stop()  # 鼠标进入工具栏 → 保持显示

    def leaveEvent(self, event):
        self._timer.start(1500)  # 鼠标离开 → 1.5秒后关闭
```

### 3.7 事件频率控制（去抖 Throttle/Debounce）

文本选择变化事件在拖拽过程中会**极高频触发**（每次鼠标移动只要选中范围改变就触发一次）。豆包必然做了去抖：

```python
# 去抖实现推荐方案（300ms）
last_event_time = 0

def on_text_selection_changed(hwnd, ...):
    global last_event_time
    now = time.time()
    if now - last_event_time < 0.3:  # 300ms 内忽略后续事件
        return
    last_event_time = now

    # 等用户松开鼠标后再处理
    threading.Timer(0.3, process_selection).start()
```

**更稳的方案**：结合 `WH_MOUSE_LL` 低层鼠标钩子，只在检测到鼠标左键释放时才读选中文字。避免拖选过程中的无用调用。

## 六、qclawTranslate 实现划词工具栏的可行路径

### 推荐方案：Python + ctypes 原生 Win32 API

**依赖：** 零额外包（仅需系统自带的 `user32.dll` + `oleacc.dll` + `UIAutomationCore.dll`）

**实现步骤：**

1. **注册 SetWinEventHook**
   - 用 `ctypes.windll.user32.SetWinEventHook` 注册 `EVENT_OBJECT_TEXTSELECTIONCHANGED`
   - 回调函数使用 `WINEVENT_OUTOFCONTEXT`（不需要 DLL，纯 Python 可行）
   - **注意**：回调函数必须运行在消息循环线程（PyQt5 主线程已具备）

2. **读取选中文字**
   - 已有 `_get_selected_text()` 雏形（`comtypes` + UIA Client）
   - 需要从事件 hook 获取 `hwnd`，用 `ElementFromHandle(hwnd)` 替代 `GetFocusedElement()`
   - 加入去抖（300ms debounce）

3. **获取坐标**
   - `IUIAutomationTextRange.GetBoundingRectangles()` 获取选中区域的屏幕矩形
   - 工具栏定位在选中文字底部/右侧

4. **弹出工具栏 Popup**
   - `QWidget` + `Qt.ToolTip | Qt.FramelessWindowHint | Qt.NoFocus`
   - `setAttribute(Qt.WA_ShowWithoutActivating)`
   - 水平布局按钮条，暗黑风格 QSS

5. **工具栏行为**
   - 自动消失定时器（5s）
   - 鼠标悬停时保持
   - 新选中时关闭旧工具栏
   - 点击按钮执行对应功能

### 兼容性风险

- 部分老旧 Win32 应用不暴露 UIA TextPattern → 兜底方案：回退到 WinEventHook 检测到后使用 `SendInput Ctrl+C`
- `SetWinEventHook` 回调必须在接收 Windows 消息的线程中 → PyQt5 `QApplication.exec()` 主线程天然满足
- `comtypes` 类型库生成需要运行一次 `GetModule('UIAutomationCore.dll')` → 打包 exe 时需要处理类型库路径

## 七、结论

> **豆包划词的核心技术是 Windows Accessibility API（SetWinEventHook + UIAutomation TextPattern），这是 Windows 提供的标准能力，不涉及逆向工程、注入或 hook 攻击，完全合规。**

qclawTranslate 可以在此基础上实现类豆包划词工具栏，技术上完全可行：
- ✅ Python + ctypes 可直接调用所有所需 Win32 API
- ✅ PyQt5 主线程天然支持 WinEvent 消息循环
- ✅ `comtypes` 已安装且 UIA 类型库已生成
- ⚠️ 主要工程量：WinEvent 回调的线程安全、去抖逻辑、Popup 窗口生命周期管理

**预计开发量：** 约 200-300 行新增代码（WinEvent hook + 去抖 + Popup 窗口类 + 坐标计算）
